Verktyg inom Test

Det finns ett Hav av olika verktyg att jobba med, Frågan är vad vill företaget. Här gäller det att var mycket tydlig från företagets sida innan man börjar investera i olika verktyg. Jag har använt mig av följande verktyg så lite uppsättning av miljön som möjligt samt att de alla är opensource

Jira:
JIRA är organiserat i separata ytor, Projekt. I varje projekt ligger Ärenden inlagda. Ett projekt är alltid kopplat till projektdeltagare som kan se och redigera de ärenden som ligger där. Ärenden är i sin tur organiserade i en hierarkisk struktur: Projekt → Epics → Ärenden → Underärenden Varje ärende innehåller en beskrivning, ansvarig handläggare, kommentarer, länkade Wiki-sidor mm. Personliga ärenden
Vill man jobba mer uppstyrt kan man också använda Scrum, då tillkommer tidsuppskattning, sprintplanering etc
I ett projekt kan man alltid visa eller söka upp ärenden i en lista eller genom sökfunktionen. För det dagliga arbetet är projektets ärenden organiserade i en eller flera Tavlor där man visuellt kan tilldela, organisera, prioritera och dokumentera ärenden sinsemellan. Kanban är den vanligaste metoden för att organisera ärenden, vi använder en enkel modell där ärendestatus följs enligt: Backlog → Aktuellt → Pågående → Klar

Microsoft Visual Studio
är en avancerad programutvecklingsmiljö från Microsoft som används till att utveckla PC-baserade applikationer för Microsoft Windows och Internetanpassade distribuerade applikationer. Jag har använt denna kontinuerligt när jag skrivit min kod i Javascript.

Cypress är BÄST. Jag kan inte förstå att man sitter fortfarande i Selenium!!:
- Tidsresa, det tar ögonblicksbilder av dina tester när du kör testerna. Detta gör att du enkelt kan hålla muspekaren över varje kommando som ägde rum i ditt test - Debuggability, Våra läsbara fel och stackspår gör felsökningen blixt snabbt.
- Omladdningar i realtid, Cypress laddas automatiskt om när du gör ändringar i dina tester. De här kommandona körs i realtid i din app.
- Automatisk väntan, Cypress väntar automatiskt på kommandon och påståenden innan du går vidare.
- Spioner, stubbar och klockor, verifierar och kontrollerar beteenden hos funktioner, serversvar eller timers.
- Konsekventa resultat, vår arkitektur använder inte Selenium eller WebDriver. Säg hej till snabba, konsekventa och tillförlitliga tester som är flingfria.
- Nätverkstrafikkontroll, enkelt kontrollera, stubba och testa edge-fall utan att din server är involverad. Du kan stoppa nätverkstrafik hur du vill.
- Skärmdumpar och videor, Visa skärmdumpar som tagits automatiskt vid misslyckande, eller videor från hela testpaketet när du kör huvudlöst.

Jag har kontiuerligt jobbat med jenkins för Stadium, satt upp många auto tester och låtit dem gå i flera, flera dagar, var 15e minut. EX: end 2 end tester, där jag söker up varan till jag betalar varan i kassan. Super bra verkktyg. - Kontinuerlig integration och kontinuerlig leverans: Jenkins kan användas som en enkel CI-server eller förvandlas till ett kontinuerligt leveranshub för alla projekt.
- Enkel installation: Jenkins är ett fristående Java-baserat program, redo att köras direkt, med paket för Windows, Mac OS X och andra Unix-liknande operativsystem.
- Enkel konfiguration: Jenkins kan enkelt ställas in och konfigureras via webbgränssnittet, som inkluderar on-the-fly felkontroller och inbyggd hjälp.
- Plugins: Med hundratals plugins i Update Center integrerar Jenkins med praktiskt taget alla verktyg i den kontinuerliga integrations- och kontinuerliga leveransverktygskedjan. - Utdragbar Jenkins kan utökas via dess plugin-arkitektur, vilket ger nästan oändliga möjligheter för vad Jenkins kan göra. - Distribuerad Jenkins kan enkelt distribuera arbete över flera maskiner, vilket hjälper snabbare att bygga, testa och distribuera över flera plattformar.

- Slack, en förkortning av Searchable Log of All Conversation and Knowledge. Jag har använt slack då jag är med i olika communities ex: Cypress, Code Institute, Appium, Java-script. Dessa har hjälpt mig oerhört mycket när ingen annan fanns att fråga.
- Slack är en bot som kopplar in sig på företagets personalsystem och varje morgon klockan åtta berättar den vilka i teamet som jobbar hemifrån, är sjuka eller har något annat inbokat.
- Offentliga kanaler låter olika grupper kommunicera utan att använda e-post eller grupp-SMS. De är öppna för alla i chatten förutsett att de först har blivit inbjudna dit. Privata kanaler gör det möjligt att ha privata konversationer mellan mindre grupper. Dessa kan användas för att dela upp större lag i deras egna respektive projekt. Direkta meddelanden låter användare skicka privata meddelanden till en specifik användare istället för en grupp.

- Github är en webbaserad och centraliserad lagring av versionshistorik för programvaruutvecklingsprojekt som använder versionshanteringssystemet Git.Github erbjuder gratis lagring men även abonnemang för extra funktionalitet - GitHub är en öppen källkod värdtjänst, som ett moln för kod. Det kommer att vara värd för dina källkodsprojekt. Det kan göra detta genom att använda git, ett revisionskontrollsystem som körs i kommandoradsgränssnittet. Det finns andra källor som liknar GitHub, inklusive BitBucket, Microsoft Team Foundation Server och mycket mer, men den stora storleken på samhället bör vara viktigt för dig om du vill att ditt projekt ska ses av så många som möjligt. Från och med 2018 rapporterade GitHub att ha mer än 28 miljoner användare, betydligt mer än sina konkurrenter. De övriga skillnaderna innebär kostnader. GitHub erbjuder endast privata repositorier mot en extra kostnad. Att använda GitHub gör det lättare att samarbeta med kollegor och kamrater och titta tillbaka på tidigare versioner av ditt arbete. Om du inte redan använder GitHub för dina kodprojekt, är det bara några av anledningarna till att överväga att göra det.
- Jag har använt github sedan dag 1 jag började med weebutveckling samt Test, allt som har med kod har jag lagt upp på Github.

- GitPod, en plattform som gör det möjligt för utvecklare att skapa utvecklingsmiljöer i moln och att koda i en webbläsare, har publicerat sin källa på GitHub. - GitPod-tjänsten använder containrar som körs i ett Kubernetes-kluster, vilket ger en flyktig utvecklingsmiljö, där utvecklare vanligtvis kodar i en webbläsare som använder Förmörkelse Theia, liknande Visual Studio Code (VS-kod) och kompatibel med dess språk-serverprotokoll - nyckeln till att driva intelligenta IDE-funktioner som syntaxbelysning och auto-fullständig - liksom de flesta av dess tillägg. - Jag har använt GiTPOD sedan 1 år tillbaka, det är riktigt bra IDE, krävs inga nedladdningar, direkt kopplad till Github, Bara klicka på den blå Gitpod knappen på Github så öppnas IDE, bara börja skriva och när du är klar, direkt kör den till Github. API:er finns överallt på Internet. Varje gång du skickar och tar emot meddelanden med en snabbmeddelandeapp eller när du kollar nyheterna på internet begärs och tas emot API-förfrågningar i bakgrunden. Några av HTTP-metoderna som är associerade med API: er är GET, POST, PUT, DELETE och PATCH. När en utvecklare skapar ett API måste det testas för dess kvalitet. Det finns många element som utgör modern API-testning, från att kontrollera funktionskrav till prestanda, tillförlitlighet och säkerhet för tjänsten. Det finns också många populära verktyg för att möjliggöra snabb API-testning, som SoapUI, RapidAPI och TestProject. Postman är ett annat sådant verktyg som kan hjälpa en API-utvecklares testprocess.

- Eftersom användningen av mobil är mer är efterfrågan på mobilapplikationer också hög. Eftersom mobilapplikationer är praktiska finns det nästan 2 miljoner appar i Android och IOS. Konvertering av aktuell programvarutestning till mobilversion kom till bilden för att skapa appar i mobilprogramvara och behovet av att testa dessa applikationer har väckt.
- Varför välja Appium? Med de många mobila automatiseringsverktygen på marknaden har Appium sin egen unika funktion som driver Android / iOS-system för att ge användarvänlig flexibilitet. Appium har sin förlängning att arbeta i inbyggda och hybridmobilapplikationer. Appium kan skrivas på vilket språk som helst och kan användas för automatiserade ändamål utan att ändra något. Och Appium-koden kan köras i olika enheter som är det bästa pålitliga och skalbara valet för automatisering av mobil. - Jag har precis börjat lära mig Appium för att kunna testa appar, detta är nåt som frågas av företagen idag. Det var vitkigt för mig att ta upp detta.
